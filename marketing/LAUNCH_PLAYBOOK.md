# Hivemind Launch Playbook

> **Goal**: Take Hivemind from 7 stars to 1,000+ in the first week of launch.

## Pre-Launch Checklist

- [ ] README is polished with hero GIF/screenshot
- [ ] `npx create-hivemind@latest` works end-to-end
- [ ] Landing page is deployed (GitHub Pages or Vercel)
- [ ] Demo video/GIF is ready (60-90 seconds)
- [ ] All social media posts are drafted (see below)
- [ ] GitHub Issues are clean, labels are set
- [ ] v1.0.0 tag is created

## Talking Points

### The Pitch (one-liner)
> "One prompt. A full AI engineering team. Production-ready code."

### The Story
> "I got tired of babysitting AI coding agents. So I built an open-source system where a PM, frontend dev, backend dev, tester, and reviewer work together — like a real engineering team. You give it one prompt, go lie on the couch, and come back to committed code."

### Differentiators vs Paperclip
> "Paperclip runs companies. Hivemind runs engineering teams. They're complementary — you could use Hivemind as the engineering department inside a Paperclip company."

### Technical Highlights
- DAG-based parallel execution (not sequential)
- Self-healing with circuit breakers
- Artifact flow between agents (no "telephone game")
- Corporate org structure (CEO/CTO/VP hierarchy)
- Multi-runtime support (Claude Code, OpenClaw, Bash, HTTP)
- Real-time dashboard with live agent streaming
